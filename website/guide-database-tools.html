🗄️ Database Tools Guide

Manage your MySQL and PostgreSQL databases with Adminer or HeidiSQL.

Available Database Tools

Stacknest provides two options for managing your databases:

🌐 Adminer (Web-Based)

Lightweight web interface for database management:

  • No installation required
  • Access from any browser
  • Works with MySQL and PostgreSQL
  • Simple, intuitive interface
  • Export/import databases

🖥️ HeidiSQL (Desktop App)

Full-featured desktop application for advanced database management:

  • Advanced query editor with syntax highlighting
  • Visual table designer
  • Data import/export tools
  • Batch operations on databases
  • Database synchronization

Accessing Database Tools

Step 1: Open Database Tab

Click "Database" in the left sidebar.

Step 2: Choose Your Tool

You'll see two options:

  • Adminer - Click to open in your browser
  • HeidiSQL - Click to launch the desktop application

Step 3: Log In

Enter your database credentials:

  • Host: localhost or 127.0.0.1
  • Username: Usually root for MySQL
  • Password: Your MySQL/PostgreSQL password
  • Database: Leave empty to see all databases

Using Adminer

📊 Database Operations

In Adminer, you can:

  • View all databases and tables
  • Create new databases
  • Create and edit tables
  • Add, edit, and delete records
  • Run SQL queries
  • Export databases as SQL files

💾 Backing Up Database

  1. Select your database
  2. Click "Export" at the top
  3. Choose format (SQL is recommended)
  4. Click "Export" to download
  5. Save the .sql file to your computer

📥 Restoring Database

  1. Create a new empty database
  2. Select the database
  3. Click "Import"
  4. Choose your .sql file
  5. Click "Execute" to restore

Using HeidiSQL

🔗 Creating Connection

When HeidiSQL opens, you need to create a connection:

  1. Click "New" in the Session manager
  2. Enter connection details (Host, Username, Password)
  3. Click "Open" to connect
  4. You're now connected to your databases

📝 Running SQL Queries

HeidiSQL has a powerful query editor:

  1. Click "Query" in the toolbar
  2. Write your SQL in the editor
  3. Highlight the query you want to run
  4. Press F9 or click "Execute"
  5. Results appear below the editor

🛠️ Design Tools

HeidiSQL includes visual design tools:

  • Table Designer: Right-click table → "Designer"
  • Visual view of columns, data types, indexes
  • Easy drag-and-drop column management
  • Graphical foreign key editor

Common Database Tasks

🆕 Create a New Database

In Adminer: Click "Database" at top left, enter name, click "Save"

In HeidiSQL: Right-click connection, select "Create new", enter name

📋 Create a New Table

In Adminer: Click "Create table", enter name and columns

In HeidiSQL: Right-click database, select "Create table", use designer

🔑 Add Index to Column

In HeidiSQL: Right-click table, select "Designer", right-click column, add index

This speeds up queries on that column

🔄 Migrate Data Between Databases

Export from source database, import to destination database

Or use HeidiSQL's synchronize feature for advanced migrations

Troubleshooting Database Connections

❌ "Connection Refused"

  • Ensure MySQL/PostgreSQL service is running (check Dashboard)
  • Verify correct host (usually localhost or 127.0.0.1)
  • Check that the service is listening on the correct port
  • Check firewall settings (may be blocking connections)

❌ "Access Denied" Error

  • Verify username is correct (default is usually "root")
  • Check password is correct
  • Ensure user has permissions on the database
  • Reset MySQL root password if forgotten

❌ HeidiSQL won't open

  • Ensure HeidiSQL is installed on your system
  • Verify database service is running
  • Check Logs tab for detailed error messages
  • Try opening Adminer instead as a web-based alternative

💡 Pro Tips

  • Regularly backup your databases to prevent data loss
  • Use meaningful names for databases and tables
  • Create indexes on columns you query frequently
  • Use HeidiSQL for complex queries and tasks
  • Use Adminer for quick simple database operations
  • Always test changes in a development database first

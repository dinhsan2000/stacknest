Service Management Guide

Manage Apache, Nginx, MySQL, PostgreSQL, MongoDB, PHP, and Redis from one dashboard.

Understanding the Dashboard

The Dashboard is where you control all your local services. Each service shows:

MySQL

Database: Relational

● Running

Port 3306

Each service displays:

  • Status: Green (running) or Red (stopped)
  • Port: Default port for the service
  • Buttons: Start, Stop, Restart
  • Checkbox: Include in "Start All" action

Starting Services

There are two ways to start services:

🚀 Start Individual Service

  1. Find the service in the Dashboard (e.g., Apache)
  2. Click the Start button
  3. Wait for status to change to ● Running

⏱️ Starting usually takes 2-5 seconds depending on the service.

▶️ Start All Selected Services

  1. Check the boxes for services you want to include
  2. Click "Start All" button at the top
  3. All checked services will start in sequence

💡 Tip: Your "Start All" preferences are saved automatically.

Stopping & Restarting

⏹️ Stop a Service

Click the Stop button. The service will gracefully shut down.

⚠️ Stopping a database may cause data loss if not properly flushed. Always stop gracefully.

🔄 Restart a Service

Click Restart to stop and immediately start the service. Use this when:

  • Configuration changes need to take effect
  • Service is acting up or stuck
  • You updated the service version

Troubleshooting Service Issues

❌ Service won't start

  • Check if the port is already in use (use Log Viewer to see errors)
  • Ensure the service is properly installed on your system
  • Try restarting Stacknest itself
  • Check system firewall settings

❌ Service crashes after starting

  • Click the Logs tab to see error messages
  • Check if configuration files are correct (Config Editor)
  • Look for missing dependencies or corrupted files
  • Try updating to the latest service version (Binaries tab)

⚠️ Port conflict

Two services trying to use the same port:

  1. Stop one of the conflicting services
  2. Or change the port in Config Editor
  3. Or use different services (e.g., Nginx instead of Apache)

💡 Pro Tips

  • Always stop services before shutting down your computer
  • Use Restart instead of Stop/Start for faster operation
  • Check the Logs tab when a service behaves unexpectedly
  • Save your "Start All" preferences to quickly start your dev environment

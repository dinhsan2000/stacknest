⚙️ Config Editor Guide

Edit service configuration files safely with syntax highlighting and automatic backups.

What is the Config Editor?

The Config Editor allows you to modify service configuration files directly within Stacknest:

  • Edit Apache, Nginx, PHP, MySQL, PostgreSQL configuration files
  • Syntax highlighting for different file types (Apache conf, Nginx conf, INI, JSON)
  • Real-time validation and error checking
  • Automatic timestamped backups before every save
  • Quick restart services after config changes

💡 All changes are automatically backed up, so you can safely experiment with configuration.

Accessing the Config Editor

Step 1: Open Config Editor Tab

Click "Config Editor" in the left sidebar.

Step 2: Choose a Service

Select a service from the dropdown menu:

  • Apache - Web server configuration
  • Nginx - Web server configuration
  • PHP - PHP.ini settings
  • MySQL - MySQL configuration
  • PostgreSQL - PostgreSQL configuration

Step 3: Select Configuration File

After choosing a service, you'll see available configuration files. Click to select and view:

  • httpd.conf - Main Apache configuration
  • nginx.conf - Main Nginx configuration
  • php.ini - PHP configuration
  • my.cnf - MySQL configuration

Editing Configuration Files

📝 Making Changes

The editor displays the configuration file with syntax highlighting:

  1. Locate the setting you want to change
  2. Click in the editor to position your cursor
  3. Make your edits
  4. The file shows unsaved changes indicator (dot in tab title)

💾 Saving Changes

Click the "Save" button to save your changes:

  • Stacknest automatically creates a timestamped backup
  • The backup is saved in your Stacknest config directory
  • You can revert to any previous backup if needed

✅ You'll see a success message confirming the save.

🔄 Applying Changes

After saving configuration changes:

  1. Return to the Dashboard
  2. Click "Restart" for the affected service
  3. The service will reload the new configuration

⚠️ Configuration changes take effect only after service restart.

Common Configuration Changes

🔌 Change Service Port

Example: Changing Apache from port 80 to 8080

Find: Listen 80 → Replace: Listen 8080

📁 Change Document Root

Example: Change where Apache looks for files

Find: DocumentRoot → Change path to your project directory

⚡ Enable PHP Modules

Example: Enable GD for image processing

Find: ; extension=gd → Remove the semicolon: extension=gd

💾 Increase MySQL Memory

Example: Increase max_connections for more simultaneous connections

Find: max_connections = 100 → Change to desired value

Reverting Configuration Changes

🔙 Access Backup Files

If you need to undo changes:

  1. Click the "Backups" button in the Config Editor
  2. View timestamped backup list for the current file
  3. Click a backup to view its contents
  4. Click "Restore" to revert to that version

Each backup shows when it was created, so you can find the right one.

Configuration Validation

⚠️ Syntax Errors

If your configuration has syntax errors:

  • The editor highlights the problem area
  • Error message explains what's wrong
  • Fix the issue and try saving again

❌ Service won't restart after config change

  • Check the Logs tab to see error messages
  • Review your configuration changes for syntax errors
  • Revert to the last working backup
  • Make changes more carefully (one setting at a time)

💡 Pro Tips

  • Always make one change at a time and test it before making more changes
  • Use Ctrl+F (Cmd+F on Mac) to search within the configuration file
  • Keep comments in configuration files to remember why you made changes
  • Review the service logs after restarting to confirm changes took effect
  • When in doubt, consult the official service documentation online

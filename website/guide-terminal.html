💻 Terminal Guide

Use the built-in terminal for command-line operations without leaving Stacknest.

What is the Built-In Terminal?

The Terminal tab provides a command-line interface directly within Stacknest:

  • Run shell commands without opening a separate terminal window
  • Navigate your project directories
  • Run build scripts, tests, and deployments
  • Interact with databases via CLI
  • Execute PHP, Node.js, Python, or other interpreters
  • All in one integrated interface

💡 No need to alt-tab between Stacknest and external terminal windows.

Accessing the Terminal

Method 1: Click Terminal Tab

Click "Terminal" in the left sidebar to open the terminal.

Method 2: Keyboard Shortcut

Press Ctrl+Shift+T (Windows/Linux) or Cmd+Shift+T (macOS) to open terminal quickly.

Using the Terminal

Once open:

  • Click in the terminal to focus it
  • Type your command
  • Press Enter to execute
  • Output appears immediately in the terminal
  • Scroll to see command history

Common Commands

📁 Navigation

pwd — Show current directory path

ls (Mac/Linux) or dir (Windows) — List files in directory

cd path/to/directory — Change directory

cd .. — Go up one directory

cd ~ — Go to home directory

📝 File Operations

cat filename — View file contents

mkdir foldername — Create new folder

touch filename — Create empty file

cp source destination — Copy file

mv source destination — Move/rename file

rm filename — Delete file

⚙️ Utility Commands

php -v — Check PHP version

php -S localhost:8000 — Start PHP built-in server

mysql -u root -p — Connect to MySQL

npm install — Install Node.js dependencies

composer install — Install PHP dependencies

clear — Clear terminal screen

🐛 Git Commands

git status — Check repository status

git add . — Stage changes

git commit -m "message" — Commit changes

git push — Push to remote

git pull — Pull from remote

git log — View commit history

Running Scripts and Tools

🚀 Running Build Tools

Run your project build commands:

  • npm run build — Build JavaScript project
  • npm run dev — Start development server
  • composer dump-autoload — Rebuild PHP autoloader
  • php artisan migrate — Run Laravel migrations

🧪 Running Tests

Execute test suites directly:

  • npm test — Run JavaScript tests
  • phpunit — Run PHP unit tests
  • pytest — Run Python tests

🔍 Checking Syntax

Validate code before deployment:

  • php -l filename.php — Check PHP syntax
  • node -c script.js — Check JavaScript syntax
  • python -m py_compile file.py — Check Python syntax

Terminal Features

📋 Copy/Paste

Work with terminal output:

  • Right-click to select and copy terminal output
  • Paste commands from clipboard
  • Use Ctrl+C to copy/interrupt commands

⌨️ Command History

Access previous commands:

  • Press ↑ Arrow key to see previous commands
  • Press ↓ Arrow key to go forward
  • Type partial command and press ↑ to search

🔄 Multiple Commands

Run multiple commands:

  • command1 && command2 — Run command2 only if command1 succeeds
  • command1 ; command2 — Run command2 regardless of command1 result
  • command1 | command2 — Pipe output of command1 to command2

⏱️ Background Process

Run long-running commands:

  • command & — Run command in background
  • Terminal stays responsive while process runs
  • Output still appears in terminal

Terminal Tips & Tricks

💡 Quick Tips

  • Type first few letters of command, then press Tab for autocomplete
  • clear — Clear terminal screen for better readability
  • alias — Create shortcuts for long commands
  • Use quotes around paths with spaces: "path/with spaces"
  • --help — Most commands show help with this flag

🔐 Security Tips

  • Don't paste unknown commands from internet directly
  • Be careful with rm command — deleted files are permanent
  • Use --dry-run flag to preview what commands will do
  • Always backup before running batch operations

Troubleshooting Terminal

❌ Command not found

  • The tool/command may not be installed
  • Check spelling of command
  • Install the missing tool (e.g., npm install)
  • Add tool to system PATH if needed

❌ Permission denied

  • You may not have permission to access the file/directory
  • Use chmod to change file permissions
  • Or run with sudo (use carefully!)

❌ Command stuck/not responding

  • Press Ctrl+C to stop the running command
  • If that doesn't work, close and reopen terminal
  • The command may be waiting for input (check the terminal)

💡 Pro Tips

  • Keep terminal open while developing for quick command execution
  • Use terminal for git operations instead of GUI clients
  • Run build and test commands from terminal to save time
  • Create aliases for frequently used commands to work faster
  • Use terminal output in logs for debugging complex issues

🔒 SSL Certificates Guide

Generate and use HTTPS certificates for secure local development.

Why Use SSL Certificates?

SSL certificates enable HTTPS (secure connections) for your local development:

  • Test your site exactly as users will see it in production
  • Test security features that require HTTPS
  • Develop with realistic browser warnings/trust indicators
  • Test API endpoints that require HTTPS only

💡 Stacknest creates a local Certificate Authority (CA) and per-domain certificates — no real CA needed.

Setting Up SSL Certificates

Step 1: Open SSL Tab

Click "SSL" in the left sidebar.

Step 2: Initialize Local CA (First Time Only)

On first use, click "Initialize CA". This creates a local Certificate Authority:

  • Generates a root certificate
  • Installs CA in your OS trust store (may show dialog)
  • Takes 10-30 seconds

✅ Once initialized, you won't need to do this again.

Step 3: Generate Domain Certificate

Click "+ Generate Certificate" and enter:

Domain Name: Enter your domain (e.g., myproject.local)

Stacknest will:

  1. Generate a private key and certificate
  2. Sign it with your local CA
  3. Store it securely in Stacknest's config directory

Step 4: Enable HTTPS in Virtual Host

Go to Virtual Hosts and:

  1. Edit your virtual host
  2. Enable Use HTTPS
  3. Select the certificate you generated
  4. Save changes

Step 5: Access Your Site

Open your browser and visit:

https://myproject.local

🔓 You might see a security warning because this is a self-signed certificate. Click "Advanced" and "Proceed" — this is expected for local development.

Managing Your Certificates

📋 View Certificates

The SSL tab shows all your generated certificates with:

  • Domain name
  • Generation date
  • Expiration date

🔄 Regenerate a Certificate

If a certificate expires or you need a new one:

  1. Delete the old certificate
  2. Generate a new one with the same domain name
  3. Update your virtual host to use the new certificate

🗑️ Delete a Certificate

Click the Delete button next to a certificate. This removes:

  • The certificate files
  • The private key

⚠️ You cannot recover deleted certificates. Make sure no virtual hosts are using it.

Understanding Browser Warnings

When you first visit an HTTPS site with a local certificate, you'll see warnings:

⚠️ "Your connection is not private"

This is normal for self-signed certificates. To proceed:

  1. Click "Advanced" or "Details"
  2. Click "Proceed to myproject.local (unsafe)" or similar
  3. The site will load normally

This warning will go away after your local CA is properly installed in your OS trust store.

✅ If the warning persists

This means the local CA wasn't properly installed in your OS. Try:

  1. Re-initialize the CA (may require admin privileges)
  2. Manually import the CA certificate to your browser
  3. Restart your browser after installing the CA

Wildcard Certificates (Advanced)

For multiple subdomains under one domain:

When generating a certificate, use a wildcard:

*.myproject.local

This certificate works for:

  • api.myproject.local
  • admin.myproject.local
  • anything.myproject.local

Certificate Expiration

Stacknest generates certificates that are valid for:

  • Domain certificates: 365 days (1 year)
  • Local CA: 10 years

💡 When a certificate is about to expire or has expired, Stacknest will notify you. Simply generate a new one for the same domain.

💡 Pro Tips

  • Use meaningful domain names (e.g., myapp.local, not localhost)
  • Generate wildcard certificates for multiple subdomains
  • Keep certificate files safely — don't share private keys
  • The local CA is only for development — never use it in production

📦 Binaries Manager Guide

Install, update, and manage service binaries with automatic version detection.

What is the Binaries Manager?

The Binaries Manager helps you manage the executable files and versions of your development services:

  • View installed service versions
  • Check for available updates
  • Download and install newer versions
  • Manage multiple PHP versions
  • Roll back to previous versions if needed

💡 Stacknest automatically detects installed services and their versions on startup.

Accessing the Binaries Manager

Step 1: Open Binaries Tab

Click "Binaries" in the left sidebar.

Step 2: Review Installed Services

You'll see a list of detected services on your system:

  • Service Name - Apache, Nginx, MySQL, etc.
  • Current Version - Installed version number
  • Status - ✅ Installed or ❌ Not installed
  • Location - Where the binary is located on your system

Step 3: Check for Updates

For each service, you can see if updates are available:

  • Green checkmark: You have the latest version
  • Blue arrow: Update available
  • Gray: Service not detected on your system

Managing Service Versions

📲 Updating a Service

To update a service to the latest version:

  1. Find the service in the Binaries Manager
  2. Click the "Update Available" button
  3. Review the new version details and release notes
  4. Click "Download & Install"
  5. Wait for the download and installation to complete
  6. Stacknest will prompt you to restart the service

⏱️ Updates typically take 2-10 minutes depending on file size and your connection speed.

🔄 Switching Between Versions

For PHP, you can switch between installed versions:

  1. Go to Binaries Manager
  2. Find PHP in the list
  3. Click "Switch Version"
  4. Select your desired PHP version
  5. Services using PHP will automatically use the new version

🗑️ Removing a Service

To uninstall a service binary from your system:

  1. Find the service in Binaries Manager
  2. Click the "Remove" button
  3. Confirm the uninstall
  4. The service will be removed from your system

⚠️ Removing a service will prevent it from running. Only remove if you no longer need it.

PHP Version Management

🐘 Multiple PHP Versions

Stacknest supports managing multiple PHP versions:

  • Detect already installed PHP versions
  • Download additional PHP versions
  • Switch between versions without reinstalling
  • Each virtual host can use its configured PHP version

🔧 Active PHP Version

The active PHP version is indicated with a star (⭐) in Binaries Manager:

  • Only one version can be active at a time
  • CLI commands use the active version
  • Web servers serve PHP using the active version by default
  • Click any version to make it the active one

📊 Checking PHP Info

For each PHP version, you can view its configuration:

  1. Click on a PHP version in Binaries Manager
  2. Click "View PHPinfo"
  3. Browser opens with detailed PHP configuration
  4. Check installed extensions, versions, settings, etc.

Version Compatibility

⚠️ Before Updating Services

Before updating a service, consider:

  • Breaking changes between versions (check release notes)
  • Application compatibility with the new version
  • Time needed for testing after update
  • Rollback plan if the update causes issues

💡 Test updates on a non-production environment first if possible.

Troubleshooting Binaries

❌ Service not detected

  • Service may not be installed on your system
  • Installation path may be non-standard
  • Try clicking "Scan for Services" to re-detect
  • Manually download and install the service

❌ Update failed

  • Check your internet connection
  • Try downloading again
  • Ensure you have enough disk space
  • Check the Logs tab for detailed error information
  • Download and install manually from the official website

❌ Service won't start after update

  • Check the Logs tab to see what error occurred
  • Verify configuration compatibility with new version
  • Reinstall the service to reset configuration
  • Consider rolling back to the previous version

💡 Pro Tips

  • Keep services updated to get security patches and bug fixes
  • Test applications thoroughly after updating services
  • Check release notes before updating for breaking changes
  • Use version switching to test your app on different PHP versions
  • Backup your configuration before major updates

📋 Log Viewer Guide

Monitor service logs in real-time to debug issues and track performance.

What is the Log Viewer?

The Log Viewer shows you real-time and historical logs from all your services:

  • Real-time log streaming as services generate output
  • Color-coded log levels (INFO, WARNING, ERROR)
  • Search logs for specific errors or messages
  • Filter logs by service
  • Save and export logs for analysis
  • Track performance and system events

💡 Logs help you diagnose issues when services fail to start or behave unexpectedly.

Accessing the Log Viewer

Step 1: Open Logs Tab

Click "Logs" in the left sidebar.

Step 2: Select Service

Choose which service's logs you want to view:

  • Apache HTTP Server
  • Nginx
  • MySQL
  • PostgreSQL
  • MongoDB
  • PHP
  • Redis

Step 3: View Logs

The log viewer displays logs in real-time:

  • New log entries appear automatically
  • Latest entries are shown at the bottom
  • Colored by log level (different colors for errors, warnings)
  • Scroll up to see older entries

Understanding Log Levels

Logs are categorized by importance level:

ℹ️ INFO

Informational messages about normal operations. You don't need to take action.

Example: "Server started successfully"

⚠️ WARNING

Something unexpected happened, but the service continues running. May indicate a problem.

Example: "Deprecated function used"

❌ ERROR

Something failed, but the service may continue running. Needs investigation.

Example: "Failed to connect to database"

🔥 CRITICAL

Severe error that may cause service to crash. Requires immediate action.

Example: "Out of memory"

Using Log Viewer Features

🔍 Searching Logs

Find specific errors or messages:

  1. Use the search box at the top of the log viewer
  2. Type keyword (error name, function name, file path)
  3. Logs are filtered to show only matching entries
  4. Matching text is highlighted in the logs

⏸️ Pause/Resume Logging

Control log streaming:

  • Click the "Pause" button to stop receiving new log entries
  • Useful for reading specific error messages before they scroll away
  • Click "Resume" to continue watching

📥 Clear Logs

Clear old logs from the display:

  1. Click the "Clear" button
  2. This removes logs from view (they're still stored on disk)
  3. Useful when logs become cluttered

💾 Export Logs

Save logs for later analysis:

  1. Click the "Export" button
  2. Choose export format (TXT or CSV)
  3. Log file is saved to your computer
  4. Share with team or save for troubleshooting

Debugging with Logs

🔧 Service Won't Start

When a service fails to start:

  1. Open the Logs tab
  2. Select the failing service
  3. Look for ERROR or CRITICAL messages
  4. Read the error message carefully — it usually explains the problem
  5. Common issues: port already in use, permission denied, config error

⚠️ Website Shows 500 Error

When your web application crashes:

  1. Check Apache or Nginx error log (look for PHP errors)
  2. Also check application log files if available
  3. Search logs for the time the error occurred
  4. Look for stack traces or error messages in the output

Common Log Messages

✅ "Server started successfully"

Service is running normally. No action needed.

⚠️ "Port 80 already in use"

Another application is using the port. Stop the other app or change the port in config.

❌ "Permission denied"

Check file/folder permissions. Service doesn't have access to required files.

❌ "Connection refused"

Can't connect to database. Ensure MySQL/PostgreSQL is running on correct port.

🔥 "Out of memory"

Service ran out of memory. Stop some services or increase available RAM.

💡 Pro Tips

  • Always check logs when something doesn't work as expected
  • Use search to find specific errors quickly
  • Export logs before uninstalling services for troubleshooting
  • Keep logs open in a separate window while developing/testing
  • ERROR and CRITICAL messages usually point directly to the problem
  • Google error messages if you can't understand them

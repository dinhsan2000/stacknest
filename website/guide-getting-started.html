Getting Started with Stacknest

Learn how to install and launch Stacknest for the first time.

Step 1: Download & Install

Visit the Download page to get Stacknest for your operating system:

  • Windows: Run the installer (.exe) and follow the setup wizard
  • macOS: Mount the DMG file and drag Stacknest to Applications
  • Linux: Make the AppImage executable or install the .deb package

💡 Installation takes about 1-2 minutes. The app requires 4GB RAM minimum.

Step 2: Launch the App

After installation, launch Stacknest:

  • Windows: Click the desktop shortcut or find it in Start menu
  • macOS: Open Applications folder and double-click Stacknest
  • Linux: Run the AppImage or use your application menu

⚠️ On first launch, Stacknest scans your system for service installations (Apache, Nginx, MySQL, etc.). This may take 30 seconds.

Step 3: Configure Your Services

Once Stacknest opens, you'll see the Dashboard with detected services:

  1. Check service status: Green = running, Red = stopped
  2. Start a service: Click the play button next to the service name
  3. Configure which services start with "Start All": Toggle the checkboxes on the left

🔧 Tip: You can change the root directory for services in Settings (bottom left icon).

Step 4: Create Your First Virtual Host

Set up a local domain name for your project:

  1. Click "Virtual Hosts" in the sidebar
  2. Click "+ Add Virtual Host"
  3. Enter domain name (e.g., myproject.local)
  4. Choose Apache or Nginx
  5. Select your project directory
  6. Click "Create"

✅ Your project is now accessible at http://myproject.local

Step 5: Access Database Tools

Manage your databases easily:

  1. Click "Database" in the sidebar
  2. Choose "Adminer" (web-based, no installation) or "HeidiSQL" (desktop app)
  3. Log in with your MySQL/PostgreSQL credentials

📊 Adminer opens in your browser. HeidiSQL launches as a desktop application.

⌨️ Keyboard Shortcuts

Ctrl+, / Cmd+, — Open Settings

Ctrl+Q / Cmd+Q — Quit Stacknest

Ctrl+Shift+T / Cmd+Shift+T — Open Terminal

💡 Pro Tips

  • Minimize to system tray to free up screen space
  • Use the Terminal tab for quick command execution
  • Check Logs tab to debug service issues
  • Install local SSL certificates for HTTPS development

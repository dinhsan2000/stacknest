Virtual Hosts Setup Guide

Create local domain names (like mysite.local) to access your projects with realistic URLs.

What are Virtual Hosts?

Virtual hosts allow you to:

  • Access your project with a real domain name (e.g., myproject.local instead of localhost:8000)
  • Test your site as it would appear in production
  • Manage multiple projects on the same server
  • Use realistic URL structures for development

💡 When you create a virtual host, Stacknest automatically updates your hosts file to point the domain to 127.0.0.1 (localhost).

Creating Your First Virtual Host

Step 1: Open Virtual Hosts Tab

Click "Virtual Hosts" in the left sidebar. You'll see a list of existing virtual hosts (if any) and an "+ Add Virtual Host" button.

Step 2: Click "Add Virtual Host"

A dialog will open asking for:

Domain Name: Enter your desired domain (e.g., myproject.local)

Web Server: Choose Apache or Nginx

Project Root: Select your project's root directory

Step 3: Configure Settings

  • For Apache: Ensure Apache is installed and running
  • For Nginx: Ensure Nginx is installed and running
  • Your project directory must contain index.php, index.html, or similar

Step 4: Create Virtual Host

Click "Create". Stacknest will:

  1. Create a virtual host configuration file
  2. Update your hosts file (may show UAC prompt on Windows)
  3. Restart the web server to apply changes

✅ Your virtual host is now ready to use!

Accessing Your Virtual Host

After creating a virtual host, access it immediately:

Open your browser and navigate to:

http://myproject.local

Replace myproject.local with your actual domain name.

💡 If the site doesn't load:

  • Ensure your web server (Apache/Nginx) is running
  • Wait 10-15 seconds for DNS to resolve
  • Clear your browser cache
  • Check the Logs tab for errors

Best Practices for Domain Names

✅ Good Domain Names

  • blog.local — Simple and descriptive
  • ecommerce-dev.local — Descriptive with hyphen
  • myapp-v2.local — Version specific
  • project-client.local — Client/project specific

❌ Avoid These

  • Using .com or .org instead of .local
  • Spaces in domain names
  • Using capital letters (use lowercase for DNS)
  • Very long domain names

Managing Existing Virtual Hosts

📝 Edit a Virtual Host

Find the virtual host in the list and click Edit to change:

  • Document root (project directory)
  • SSL certificate settings

🗑️ Delete a Virtual Host

Click the Delete button. This will:

  • Remove the virtual host configuration
  • Update your hosts file (remove the entry)
  • The domain will no longer resolve to your server

⚠️ Deleted virtual hosts cannot be recovered. Make sure you're deleting the right one!

🔄 Update a Virtual Host

If you move a project directory:

  1. Edit the virtual host
  2. Update the project root path
  3. Save the changes

🔒 Adding HTTPS to Virtual Hosts

Want to use HTTPS (secure) with your virtual host?

  1. Go to the SSL Certificates tab
  2. Generate a certificate for your domain (e.g., myproject.local)
  3. Edit your virtual host and enable HTTPS
  4. Access your site at https://myproject.local

📚 See the SSL Certificates guide for detailed instructions.

Troubleshooting Virtual Hosts

❌ Domain doesn't resolve (ERR_NAME_NOT_RESOLVED)

  • Check that the virtual host was created successfully
  • Restart your browser or clear DNS cache: ipconfig /flushdns (Windows)
  • On Linux: sudo systemctl restart systemd-resolved
  • Verify the hosts file was updated correctly

❌ 404 Not Found when accessing the domain

  • Ensure the document root path is correct
  • Check that index.php or index.html exists in the root directory
  • Verify the web server (Apache/Nginx) is running
  • Check server logs for configuration errors

❌ Permission denied (403 Forbidden)

  • Check file permissions on the project directory
  • Ensure the web server user has read permissions
  • On Windows: right-click folder → Properties → Security
  • On Mac/Linux: use chmod to adjust permissions
